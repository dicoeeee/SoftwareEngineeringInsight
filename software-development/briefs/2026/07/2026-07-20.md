## 核心信号

### `sd-20260719-cline-team-run-role-failure-semantics` — Cline 修复 Team Run 的角色工具暴露与失败状态传播

- 主分类：开发协作与工程知识上下文
- 横向标签：Multi-agent、Team Run、Role Capability、Failure Semantics、Retry、Observability、Tool Registry
- 自主程度：L3
- 证据状态：已证实事实
- 来源优先级：P1
- 发生了什么：Cline 发布 Team Runs 后，修复 Teammate 可见但运行时禁止的 Spawn Tool，以及模型流错误被 Orchestrator 误记为 Completed、从而绕过 Retry 的问题。
- 为什么重要：Agent Team 的 Tool Registry、授权层和任务状态机必须共享同一角色与终止语义，否则会同时浪费推理预算并产生虚假成功。
- 对既有认知的影响：把既有“后台 Agent 生命周期”和“Subagent 递归上限”信号进一步具体化为角色级最小工具集与协议错误传播契约。
- 尚待确认：真实失败率、重试幂等性、依赖图的部分完成语义与 Orchestrator 重启恢复。
- 支持来源：[Cline SDK v0.0.65](https://github.com/cline/cline/releases/tag/sdk/sdk/v0.0.65)、[remove team_spawn_teammate from teammate tools](https://github.com/cline/cline/pull/12371)、[handle teammate model stream errors](https://github.com/cline/cline/pull/12370)

### `sd-20260719-claude-bun-rust-runtime` — Claude Code 分发件出现 Bun Rust Runtime 的独立实机证据

- 主分类：编码与代码修改
- 横向标签：Runtime、Bun、Rust、Binary Inspection、Supply Chain、Performance Claim
- 自主程度：待核验
- 证据状态：早期信号
- 来源优先级：P0
- 发生了什么：Simon Willison 在一台 macOS arm64 设备的 Claude Code 二进制中检出 `Bun v1.4.0` 与大量 `.rs` 路径，独立确认分发件包含 Bun Rust Port；版本范围和约 10% Linux 启动提升仍只是被转述的厂商主张。
- 为什么重要：Agent Harness 的 Runtime 会影响启动、稳定性、分发与供应链审计，实际发布制品可成为验证产品声明的证据层。
- 对既有认知的影响：把观察面从产品功能与控制面扩展到底层 Runtime 与制品级验证，不把单机结果外推到全部平台。
- 尚待确认：版本/平台覆盖、Rust Port 的实际职责边界，以及独立跨平台性能与稳定性基准。
- 支持来源：[Claude Code uses Bun written in Rust now](https://simonwillison.net/2026/Jul/19/claude-code-in-bun-in-rust/)
