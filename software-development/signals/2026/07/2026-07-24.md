### 信号

WorkBuddy Bench 以真实 commit、PR 或业务场景反向构造并重写任务，公开任务目录、Docker 环境、评测 Harness（把模型、工具和环境接入同一任务的执行框架）、测试和参考解法。其 Code、Web、Office、Security 子集在 CodeBuddy Code 与 Claude Code 上按统一协议运行；Code 面向补丁与任务测试，Web 面向前端任务结果，Office 面向目标制品正确性，Security 面向安全任务结果，各子集评分工具不同，论文不提供跨子集总平均或换算公式。

### 为什么重要

模型、Harness、任务和评分共同决定 Agent 结果。该项目把可复跑的工作单元、环境隔离与轨迹记录作为评测接口，使团队能够将“模型效果”拆解为可检查的工程变量，而非只比较排行榜名次。

### 尚待确认

- 独立团队对任务、Docker 环境与双 Harness 的复跑结果。
- 任务重写对网页检索污染的抵抗强度及数据集版本演进。
- 各 Harness 的配置、重试和工具权限是否可比，原文未披露的统一推理预算和逐任务重复次数，以及数据许可的长期边界。

### 信号

GitHub MCP Server 支持下一版 MCP（Model Context Protocol，模型上下文协议）规范：核心不再依赖 session 和 `initialize` 顺序，服务器删除 Redis session 读写，以 HTTP 请求头提供日志和 secret scanning（密钥泄漏扫描）所需值。官方同时提供 conformance tests（一致性测试套件），用于检查客户端与服务器的协议一致性。

### 为什么重要

Agent 工具服务的可扩展性和可验证性不必依赖一次初始化后保留的协议会话。无状态请求、并行握手和一致性测试为多客户端工具接入提供了更清晰的协议层边界，但鉴权、长任务和业务副作用仍应作为独立控制面设计。

### 尚待确认

- 规范正式生效后的客户端互操作性与回退行为。
- 高并发下认证、限流和长任务进度的状态语义。
- conformance tests 对权限隔离、secret scanning 和业务工具副作用错误的实际覆盖。
