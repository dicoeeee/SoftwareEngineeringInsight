### 信号

Simon Willison 认为，LLM Agent 不应被视为项目的 Directly Responsible Individual（DRI）：Agent 可以执行工作，但机器无法像人一样对行动承担组织责任，因此项目的最终责任仍应明确归属于人。

### 为什么重要

这为研发 Agent 治理补充了一个比“逐次人工审批”更上层的组织原则：即使 Agent 获得更高自主程度，也应把执行者、审批者和最终责任人分开建模。对产品和流程设计而言，任务记录、审计链和异常升级机制需要始终能定位到人类责任主体，而不能把 Agent 名称当作责任归属。

### 尚待确认

- 企业是否已把“人类 DRI + Agent 执行角色”落实到研发任务、代码所有权或审计系统中。
- 该原则在后台 Agent、多 Agent 团队和跨系统长任务中应如何映射到审批、移交与事故责任。
