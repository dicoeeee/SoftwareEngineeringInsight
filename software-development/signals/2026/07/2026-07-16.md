### 信号

Simon Willison 报告，Grok CLI 可能把其运行目录整体上传至 xAI 使用的 Google Cloud Storage；一名用户称上传内容包括 SSH Key、密码库和个人文件。该影响范围仍是早期信号，不能据此确认所有版本、用户或目录都会上传。Simon 同时记录了较确定的后续动作：xAI 已停用上传，称从 7 月 12 日起默认关闭 Coding 数据保留并删除此前数据，并在同日以 Apache 2.0 开源 Grok Build；他检查开源代码时发现 GCS 上传代码仍在，但入口返回 disabled，且仓库仅有一次初始提交、没有可供追溯的历史。

### 为什么重要

“本地 CLI”不等于“数据只留在本地”。Coding Agent 的安全边界除文件写入和 Worktree 隔离外，还必须覆盖默认网络出口、收集目录、上传前预览与授权、服务端保留期限和可审计删除。若 Harness 的开源快照缺少提交历史，也很难独立判断上传逻辑何时引入、何时停用及线上版本是否与代码一致。

### 尚待确认

- 受影响版本、用户数量、具体收集路径、触发条件和现实上传规模。
- 上传前是否存在明确授权或目录预览，以及网络端点、加密、访问控制与服务端删除审计方式。
- 厂商线上版本与开源快照的对应关系，以及缺失提交历史所遮蔽的变更过程。

### 信号

Anthropic 在 Claude Code `v2.1.211` 的官方 Release 中确认并修复两类审批完整性问题。第一，转发到 Chat Channels 的 Permission Preview 未中和双向文本覆盖字符、零宽字符和形似引号字符，工具输入因此可能改变审批消息的视觉呈现。第二，Auto Mode 此前可能覆盖 `PreToolUse` Hook 对非 Sandbox Bash 返回的 `ask` 决策；修复后，Hook 的 `ask` 会成为决策下限并强制进入询问。`v2.1.214` 又集中修复了单段 `dir/**` Allow Rule 错误匹配任意深度同名目录、Windows PowerShell 5.1 Permission Check Bypass、Bash 文件描述符重定向解析差异、zsh 变量下标与 Modifier、超长命令、部分 `help`/`man` 形式、Docker/Podman Daemon Redirect 参数，以及远程 Session 在本地确认完成前继续执行等问题；超过 10,000 字符的命令现改为始终询问。这些是官方确认的产品缺陷及修复，不应外推为已发生攻击、安全事件或正式漏洞通告。

### 为什么重要

Agent 的授权控制不只取决于是否弹出审批框，还取决于审批内容能否被忠实呈现，以及多个决策层合并时更严格的策略是否保持优先。多渠道审批与自动模式会增加控制链路；若 Unicode 呈现和 Hook 优先级没有明确不变量，用户可能批准的并不是实际执行内容，组织策略也可能在无感知下被降级。

### 尚待确认

- 受影响版本范围、现实触发频率，以及其他审批渠道和字符组合是否采用同样的规范化处理。
- 不同 Tool、Mode 和多重 Hook 组合下的完整决策优先级与回归测试覆盖。
- 修复后界面是否同时提供原始输入、转义视图或其他可审计表示。
- 路径匹配、不同 Shell AST、远程确认与容器参数的修复是否覆盖嵌套引用、别名、跨平台编码及其他等价语法。

### 信号

Claude Code `v2.1.211` 同时修复多项后台 Agent 生命周期缺陷：显式指定模型的 Subagent 在 Resume 或后续跟进时可能退回父 Agent 模型；重新打开已停止的后台 Session 可能在相同 ID 下启动空白对话；用户终止的后台 Agent 可能自动重生并重新执行旧 Prompt；结果上报可能在任务仍运行时给出虚构的完成结果，现改为报告仍在运行并等待真实完成。该版本还增加可选开关，可把 Subagent 的文本与 Thinking 转发到 `stream-json` 输出。`v2.1.216` 又修复 Resume 后未恢复原 Agent Prompt 与 Tool Restrictions 的问题，把恢复不变量从 Session ID 和模型扩展到角色身份与能力集合。

### 为什么重要

并行 Agent 的可靠性取决于控制面状态，而不仅是单次生成质量。Session ID、模型选择、Kill、Resume 和 Completion 必须是稳定且可审计的不变量；否则系统会在用户以为已经停止、恢复或完成时执行不同动作，造成重复副作用、成本失控和错误自动化决策。流式暴露子 Agent 过程有助于可观测性，也同步引入数据策略与输出开销问题。

### 尚待确认

- 进程崩溃、网络中断和跨设备恢复时，Kill、Resume、Session ID 与模型路由语义是否一致。
- 恢复原 Prompt 与 Tool Restrictions 的回归测试、受影响版本范围和跨设备一致性尚未公开。
