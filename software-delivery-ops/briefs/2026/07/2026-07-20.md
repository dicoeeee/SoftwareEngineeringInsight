# 2026-07-20 软件交付与运维 Agent 专题简报

## 今日判断

## 核心信号

### `sdo-20260719-gh-aw-credit-exhaustion` — gh-aw 用预算、确定性工作集与错峰调度治理共享 AI Credits 耗尽

- 主分类：流水线编排与调度
- 横向标签：GitHub Agentic Workflows、AI Credits、预算、确定性切片、fan-out、调度
- 自主程度：L3 受控执行
- 证据状态：早期信号
- 来源优先级：P1
- 发生了什么：多个定时 Agentic Workflows 竞争账户级 `1000-AIC` 池并在有效工作前以 403 失败；合并修复同时加入每次/每日额度、turn 上限、较低成本模型、确定性 package slice、禁止 subagent fan-out 与错峰调度。
- 为什么重要：多 Agent 调度的可靠性需要账户级容量与工作集控制，不能只给每条任务单独限额。
- 对既有认知的影响：在既有沙箱与副作用治理之外，新增了共享推理资源被内部 Agent 批处理耗尽这一生产失败模式。
- 尚待确认：阈值依据、全局排队和优先级、质量退化，以及发版后的失败率与成本变化。
- 支持来源：[Reduce scheduled workflow AI-credit burn and stagger the midnight batch](https://github.com/github/gh-aw/pull/46510)

### `sdo-20260716-aws-devops-agent-failure-drill` — AWS 将故障注入、事件关联与 Agent 调查组成可重复演练闭环

- 主分类：故障诊断、事件响应与自动修复
- 横向标签：AWS DevOps Agent、Kiro、故障演练、EventBridge、Prometheus、Grafana、EKS、Aurora、ElastiCache、MSK
- 自主程度：L3 受控执行
- 证据状态：早期信号
- 来源优先级：P1
- 发生了什么：Kiro 在人工确认 runbook 后向四类 AWS 服务注入故障；事件或指标 webhook 触发 DevOps Agent，相关事件被合并进同一调查，并在 `COMPLETED` 后回收摘要。
- 为什么重要：故障演练可同时检验基础设施恢复和 Agent 的触发、关联、完成、交付语义；提前取消会永久失去摘要这一限制也必须进入断言。
- 对既有认知的影响：从“Agent 能调查事故”的产品能力，推进到可重复注入与结果轮询的评测工作流；四个结果仍只是厂商演示。
- 尚待确认：重复试验、准确率、webhook 鉴权、最小权限、回滚，以及与 FIS stop condition 的稳定集成。
- 支持来源：[AWS DevOps Agent 实战：如何使用生成式 AI 加速故障演练](https://aws.amazon.com/cn/blogs/china/aws-devops-agent-how-to-using/)

### `sdo-20260708-pagerduty-risk-gated-agent-merge` — PagerDuty 用风险分组与可撤回策略积累 Agent 自动合并证据

- 主分类：变更接入、代码评审与合并准入
- 横向标签：PagerDuty、Coding Agent、风险分组、双阶段分类、自动合并、可撤回授权
- 自主程度：L1 辅助洞察
- 证据状态：早期信号
- 来源优先级：P1
- 发生了什么：PagerDuty 把十类工作与五级影响组合为 50 个风险组，用实现前 plan 和完成后 diff 两次分类，只允许满足样本量与无修改批准率阈值的组在未来获得自动批准资格。
- 为什么重要：自主权被放在可修改、可缩小和可撤回的 policy 中，并计划用 revert、failed deployment、rollback 与 incident 做后验复查。
- 对既有认知的影响：这是 `2026-07-08` 的历史补录；系统明确尚未执行 merge decisions，因此不能把证据收集框架写成已启用自动合并。
- 尚待确认：类别定义、分类器准确率、组内阈值、选择偏差，以及真实启用后的 queue、override 与事故归因。
- 支持来源：[Before AI Ships Code, Show Me the Receipts](https://www.pagerduty.com/eng/before-ai-ships-code-show-me-the-receipts/)
