# 2026-07-17 软件交付与运维 Agent 专题简报

## 今日判断

本轮出现两条互补路径：Google SRE 公开了从推理、动态授权、确定性执行到紧急停止和持续评估的 L3 控制面；Grafana 则把 L1 调查助手扩展到 30+ 数据源，并复用现有 RBAC、审计和数据源权限。既有 GitLab、gh-aw 与 OpenSRE 更新进一步表明，Agentic 交付和运维的竞争点正从“能否生成动作”转向“默认拒绝、风险分级、身份/令牌隔离、意图门禁、持续评估与可恢复运行”。Google 的效果数字属于公司自报，Grafana Investigations、GitLab 部分 Flow 与 gh-aw 仍为 Preview/Beta/Pre-release，不应上升为成熟行业事实。

## 核心信号

### `sdo-20260528-google-sre-actuation-control-plane` — Google SRE 将 Agent 推理与高风险执行拆成渐进授权控制面

- 主分类：故障诊断、事件响应与自动修复
- 横向标签：AI Operator、Actuation Agent、Progressive Authorization、dry-run、Agent Identity、Circuit Breaker、Red Button、Nightly Evals
- 自主程度：L3 受控执行
- 证据状态：厂商主张
- 来源优先级：P1
- 发生了什么：Google SRE 将 reasoning Agent 与确定性执行网关分离，以 dry-run、风险评分、动态 L3→L2 降权、独立 Agent 身份、速率限制、结果验证和全局 Red Button 约束缓解动作，并从人工事故轨迹构建持续评估数据。
- 为什么重要：它把高风险 SRE Agent 所需的授权、执行、中断和验证机制放进同一控制面，而不是依赖提示词自律。
- 对既有认知的影响：为 PagerDuty 并行调查、Google SecOps Agents 与 Harness L3 治理补充“风险驱动的动态降权”和独立 actuation 层；因发布时间较早，按历史补录处理。
- 尚待确认：生产覆盖、效果指标、L3 动作清单、风险阈值、Red Button 恢复和评估协议。
- 支持来源：[Google SRE whitepaper](https://sre.google/resources/practices-and-processes/ai-engineering-reliable-operations/)、[Google Cloud engineering blog](https://cloud.google.com/blog/products/devops-sre/how-google-sre-is-using-agentic-ai-to-improve-operations/)

### `sdo-20260714-grafana-cross-source-assistant` — Grafana Assistant 将跨 30+ 数据源调查纳入统一权限边界

- 主分类：故障诊断、事件响应与自动修复
- 横向标签：Grafana Assistant、Observability、跨源关联、RBAC、Audit、Multi-agent、Infrastructure Memory
- 自主程度：L1 辅助洞察
- 证据状态：早期信号
- 来源优先级：P1
- 发生了什么：Grafana Assistant 通过原生 data source architecture 在一次调查中读取、查询和关联 30+ 数据源，并继续受既有 RBAC、审计和数据源权限约束；Investigations Public Preview 提供 multi-agent、infrastructure memory 和结构化报告。
- 为什么重要：跨栈事故调查不必另建一套旁路连接器身份，权限边界可以复用既有可观测平台控制面。
- 对既有认知的影响：把调查 Agent 从单平台信号扩展到跨源关联，但仍是 L1 辅助，不等同于自动 RCA 或修复。
- 尚待确认：跨源准确率、查询深度、成本、memory 治理、生产采用和 self-managed 支持。
- 支持来源：[Grafana product blog](https://grafana.com/blog/stop-switching-tools-to-find-answers-grafana-assistant-now-works-across-30-data-sources/)、[Grafana Assistant documentation](https://grafana.com/docs/grafana-cloud/machine-learning/assistant/introduction/)
