# 2026-07-18 软件交付与运维 Agent 专题简报

## 今日判断

本轮最值得关注的变化一正一反：Datadog 把成本与可观测数据并入 FinOps 调查，说明运维 Agent 的上下文正从故障扩展到支出归因；OpenSRE 则从 L1 RCA 跨到可直接改变 GitHub Issue/PR 与合并状态的 L3 写操作，却没有逐次 approval gate。GitHub code review 和 `gh-aw` 的更新进一步表明，head-branch 指令、custom setup、self-hosted runner、凭据来源和自定义 provider 都会改变 Agent 的真实安全边界，不能只看“有 firewall/沙箱”的产品标签。

## 核心信号

### `sdo-20260717-datadog-cloud-cost-skill` — Datadog Bits Chat 将成本与可观测数据并入 FinOps 调查

- 主分类：云、Kubernetes 与平台运维
- 横向标签：Bits Chat、FinOps、Cloud Cost、AI Cost、成本异常、可观测关联、预算、权限
- 自主程度：L1 辅助洞察
- 证据状态：已证实事实
- 来源优先级：P1
- 发生了什么：Cloud Cost skill 可调查 cloud、SaaS、AI、custom 和 Datadog 成本，识别团队、服务、账号、区域和资源驱动项，把成本变化与 CPU、memory、request volume、storage 等指标关联，并比较实际/预测支出与预算；写入调查记录另需 Notebook 权限。
- 为什么重要：Agent 开始在同一上下文里连接“系统怎么运行”和“为什么变贵”，把 FinOps 从独立报表推进为可追问的运维调查。
- 对既有认知的影响：呼应 Gemini Cloud Assist 的持续 FinOps 与 Grafana 的跨源调查，但仍是 L1 辅助，不能推断为自动优化或成本治理闭环。
- 尚待确认：归因准确率、跨云口径、数据时效、权限继承、审计轨迹和独立效果。
- 支持来源：[Answer any cost question faster with the Cloud Cost skill in Bits Chat](https://www.datadoghq.com/blog/cloud-cost-skill-bits-chat/)

### `sdo-20260718-opensre-github-chat-writes` — OpenSRE 在 Chat 中开放无单次审批的 GitHub 写操作

- 主分类：变更接入、代码评审与合并准入
- 横向标签：OpenSRE、GitHub CLI、ChatOps、Issue/PR mutation、无单次审批、命令拒绝列表、Token 脱敏、任意 API
- 自主程度：L3 受控执行
- 证据状态：早期信号
- 来源优先级：P1
- 发生了什么：OpenSRE 7.18 的 Action Agent 可从 Chat 执行 Issue/PR 创建、编辑、评论、关闭和合并，工具显式设置 `requires_approval=False`；实现阻止一组高风险顶层命令并脱敏 token，但允许不限制 method/endpoint 的 `gh api`。
- 为什么重要：能力从调查建议跨到会改变协作与合并状态的副作用执行，真实爆炸半径由 token 权限、会话授权和工具约束共同决定。
- 对既有认知的影响：不改变既有异步 RCA 信号的 L1 定位，而是形成独立的 L3 写操作信号；denylist 不能被当作完整的权限模型。
- 尚待确认：`gh api` 绕过面、默认 token scopes、多仓库边界、强制预览/确认、审计、幂等与回滚。
- 支持来源：[OpenSRE 0.1.2026.7.18](https://github.com/Tracer-Cloud/opensre/releases/tag/v0.1.2026.7.18)、[PR #4057](https://github.com/Tracer-Cloud/opensre/pull/4057)
