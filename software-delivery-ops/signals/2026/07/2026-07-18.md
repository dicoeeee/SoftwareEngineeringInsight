### 信号

Datadog 发布 Bits Chat 的 Cloud Cost skill。它可从成本异常或自然语言问题启动调查，覆盖 cloud、SaaS、AI、custom 与 Datadog 成本；识别 team、service、account、region、resource 等支出驱动项，比较实际或预测支出与预算，并把成本变化与 CPU、memory、request volume、storage size 等可观测指标关联。用户可继续追问并把调查写入 Datadog Notebook；读取成本数据需要 Bits Chat 与 Cloud Cost Management 权限，创建或更新调查记录还需要相应 Notebooks 权限。官方能证明这些产品能力和权限前提，不能证明“within minutes”等效果表述。

### 为什么重要

分析判断：运维 Agent 的上下文正在从故障与发布遥测扩展到成本归因。同一工作流同时读取账单维度和运行指标，能把“支出上涨”继续追到具体团队、服务、资源或使用量变化；这与 Gemini Cloud Assist 的持续 FinOps 和 Grafana 的跨源调查形成呼应，但当前仍是 L1 调查辅助，不能据此推断它能自动执行成本优化。

### 尚待确认

- 成本归因和“rate versus usage”判断的准确率、延迟、查询成本及跨云口径一致性。
- AI 成本按 provider、model、user 与 API key 规范化时的缺失数据、时效和租户隔离边界。
- Notebook 写入是否保留完整来源、查询轨迹和权限继承，以及错误归因的纠正机制。
- 官方关于答复速度和优化效果的表述缺少独立验证。

### 信号

OpenSRE 0.1.2026.7.18 为 Action Agent 加入认证的 `github_cli` 工具，可从交互式 Chat 执行 GitHub 读取与写入，包括创建、编辑、评论和关闭 Issue/PR、合并 PR，以及调用 `gh api`。tag 中工具声明将 `side_effect_level` 标为 `mutating`，同时设置 `requires_approval=False`。运行器阻止 `auth`、`extension`、`workflow`、`run`、`secret`、`codespace`、`ssh-key`、`gpg-key`、`config` 等顶层命令，限制 1–120 秒超时，并在 stdout/stderr 中替换配置 token；但 `gh api` 仍是被明确允许的任意 API 通道，代码没有对 HTTP method 或 endpoint 再做限制。

### 为什么重要

分析判断：这不是既有 OpenSRE 异步 RCA 服务的普通渠道更新，而是从 L1 调查辅助跨到可改变 GitHub 协作与合并状态的 L3 执行面。顶层命令拒绝列表和 token 脱敏能阻止一部分直接凭据泄漏或高风险入口，但允许任意 `gh api` 且无逐次审批意味着真实爆炸半径主要由 token 权限、Action Agent 的意图理解与上游会话授权决定；不能把“有 denylist”直接视为完整的副作用隔离。

### 尚待确认

- `gh api` 是否可通过 REST/GraphQL 方法触达被顶层拒绝列表意图阻止的 workflow、secret 或其他高风险变更；本轮未执行任何利用或外部写操作。
- GitHub token 的默认 scopes、仓库边界、短期性、撤销与组织级策略，以及多仓库 `-R`/API endpoint 的限制。
- Chat 中是否存在强制的人类意图确认、操作预览、幂等键、回滚或高风险动作分级；tag 中实现未展示单次 approval gate。
- Public Alpha 的提示注入防护、审计记录、误操作率和真实生产使用表现。
