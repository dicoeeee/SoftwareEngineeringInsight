### 信号

kagent v0.10.0-beta10 将 A2A（Agent-to-Agent）只读共享从 HTTP 动词判断改为 JSON-RPC 方法级授权：`ListTasks`、`GetTask`、订阅与读取 Push 配置可通过，`SendMessage`、取消任务和写入 Push 配置等方法由 `requireWritableShare` 拒绝。关联 PR 说明此前所有 A2A 请求都通过 POST，传输层的非 GET 拦截使默认只读共享无法读取任务。该版本还修复 OpenAI Agents SDK 流式 Responses 被低层 `OpenAIInstrumentor` 重复插桩后崩溃的问题，只在追踪或日志启用时安装插桩，并由 Agents 专用插桩承担运行时观测。

### 为什么重要

分析判断：协议级共享权限应围绕操作语义而不是传输载体建模，否则 JSON-RPC 这类复用 POST 的协议会把读能力误封或迫使系统放宽整个入口。遥测修复同时表明观测组件属于 Agent 运行路径的一部分，必须验证“关闭后零影响”和“启用后不改变流式语义”，不能把插桩视为旁路设施。

### 尚待确认

- A2A v0/v1 的所有方法、未来扩展方法和 Push 回调是否都默认失败关闭，方法清单如何避免版本漂移。
- 只读 share token 的租户、会话、任务与返回字段范围，以及订阅流是否可能泄露跨会话信息。
- 遥测修复在 OpenAI 流式、工具调用、取消、重试与其他 provider 组合下的回归覆盖和性能开销。
- beta10 在真实 Kubernetes 集群中的兼容性、升级路径和生产可靠性。
