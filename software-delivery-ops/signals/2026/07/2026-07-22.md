### 信号

Harness 发布 Agent Development Lifecycle（Agent DLC）产品组合，把 Agent 的离线评测与流水线质量门禁、Amazon Bedrock AgentCore 和 Google Agent Runtime 部署、渐进式流量切换、OPA 与审批、运行时 prompt（提示词）/model（模型）配置、资产目录、AIBOM、运行时防火墙和跨会话 AgentTrace 放入同一交付平台。官方材料还描述生产失败可回流为评测用例，prompt（提示词）与 model（模型）配置可独立版本化和回滚；这些产品行为和可用性均来自厂商材料，尚无独立生产验证。

### 为什么重要

分析判断：既有 Harness 信号证明 Agent 可以作为受治理的流水线执行步骤，本次发布进一步把“Agent 自身是被交付的软件系统”具体化为可评测、可准入、可渐进部署、可回滚和可追踪的发布对象。它说明传统 CI/CD 控制面正在吸收非确定性系统所需的新发布资产与反馈回路，但产品组合的整合度、门禁效果和运行时治理能力仍待真实环境验证。

### 尚待确认

- 各模块的实际 GA/Preview 状态、地域和许可边界，以及“rolling out now”对应的客户可用范围。
- AI Evals 的数据集管理、评分器一致性、阈值校准、失败阻断与在线样本回流的隐私边界。
- Agent Deployments 对 Bedrock AgentCore 与 Google Agent Runtime 的回滚、失败补偿、会话迁移和跨组件发布一致性。
- AI Configs 在不重新部署时修改 prompt（提示词）、model（模型）与参数的身份、审批、版本绑定和审计语义。
- AgentTrace、AIBOM、AI Firewall 和 Asset Catalog 的覆盖完整性、性能开销、误报漏报及独立生产效果。
