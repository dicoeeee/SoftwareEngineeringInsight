# Software Engineering Agent Insight

本仓库持续记录软件研发与 IT 运维领域的 Agent 技术洞察，覆盖从需求、设计、编码到 Commit/PR 之后的软件交付、生产运维与安全运营。

内容由 Scheduled 工作流定期汇总，重点关注：

- Agent 产品与开源能力、技术方向和应用场景；
- 系统架构、Harness、工具调用、工作流、沙箱、权限、验证与审计机制；
- 真实工程实践、公开评测、量化效果、失败案例和技术限制；
- 已证实事实、早期信号、厂商主张和分析判断之间的证据边界。

## 内容结构

- [`software-development/`](./software-development/)：Commit 之前的软件开发 Agent。
- [`software-delivery-ops/`](./software-delivery-ops/)：从 Commit/PR 开始的软件交付与生产运维 Agent。
- [`reports/weekly/`](./reports/weekly/)：跨专题周报。

每个专题包含基线、日度简报和事件级信号记录。同一事件沿用稳定信号 ID；新增证据追加到原记录并保留更新历史。标记为 `excluded` 的记录仅用于保留人工更正审计，不进入后续趋势统计。

## 边界说明

本仓库不追求穷尽所有产品或资讯，也不收录没有具体技术机制或工程实践的组织管理、岗位责任、文化伦理与宏观社会议题。厂商发布和效果数字不会自动视为独立验证结果。

内容用于技术研究与趋势观察，不构成采购、安全、法律或投资建议。
